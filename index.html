Key Information

Project Guidelines

Success Criteria

  • All users active on M365
  • Email flowing through Exchange Online
  • MFA enabled for all accounts
  • Zero data loss during migration
  • All files migrated to OneDrive/SharePoint
  • Staff trained on core M365 apps

Risk Mitigation

  • Full backup before any migration step
  • Pilot group of 3–5 users before full rollout
  • 48–72 hour hypercare post-cutover
  • Rollback plan documented for each phase
  • Daily status updates during active migration

Communication Plan

  • Weekly status meetings with client leadership
  • Dedicated Teams channel for real-time updates
  • Email notifications at each phase milestone
  • 24/7 emergency contact during go-live
  • Post-migration survey for user feedback
