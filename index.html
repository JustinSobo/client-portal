Project Phases

Click any checkbox to track progress as tasks are completed.

📋 Phase 1: Discovery & Assessment
Week 1
  • Inventory current environment (email provider, file storage, devices, apps)
  • Gather user list (names, roles, departments, existing email addresses)
  • Identify licensing needs (M365 Business Basic / Standard / Premium)
  • Confirm domain ownership
  • Stakeholder kickoff call with client leadership
🔐 Phase 2: Tenant Setup & Identity
Week 2
  • Provision M365 tenant and configure Entra ID
  • Add and verify custom domain
  • Create user accounts in Entra ID
  • Set up security groups and role assignments
  • Configure MFA and security defaults
  • Set password policies and self-service password reset
📧 Phase 3: Email & Data Migration
Weeks 2–3
  • Determine current mail platform (Gmail, on-prem Exchange, other)
  • Run mailbox migration (Migration Manager or BitTitan)
  • Migrate calendars, contacts, and shared mailboxes
  • Migrate existing files to OneDrive / SharePoint
  • Set up shared drives and team sites
🚀 Phase 4: M365 App Deployment
Week 3
  • Deploy Microsoft Teams, Outlook, and Office apps
  • Configure Teams channels by department
  • Set up SharePoint for internal docs and SOPs
  • Train users on M365 basics (Teams, Outlook, OneDrive, SharePoint)
🛡️ Phase 5: Security & Compliance
Week 4
  • Apply Conditional Access policies in Entra ID
  • Configure Defender for Business (if licensed)
  • Set up DLP policies for sensitive data
  • Enable audit logging and sign-in monitoring
  • Review and harden admin accounts
🎯 Phase 6: Go-Live & Hypercare
Weeks 4–5
  • Cutover DNS (MX records pointed to M365)
  • Monitor sign-ins and mailflow for 48–72 hours
  • Provide dedicated support channel for client staff
  • Address any post-migration issues
  • Decommission old systems after validation

Key Information

🎯 Success Criteria

  • All users active on M365
  • Email flowing through Exchange Online
  • MFA enabled for all accounts
  • Zero data loss during migration
  • All files migrated to OneDrive/SharePoint
  • Staff trained on core M365 apps

⚠️ Risk Mitigation

  • Full backup before any migration step
  • Pilot group of 3–5 users before full rollout
  • 48–72 hour hypercare post-cutover
  • Rollback plan documented for each phase
  • Daily status updates during active migration

📞 Communication Plan

  • Weekly status meetings with client leadership
  • Dedicated Teams channel for real-time updates
  • Email notifications at each phase milestone
  • 24/7 emergency contact during go-live
  • Post-migration survey for user feedback
